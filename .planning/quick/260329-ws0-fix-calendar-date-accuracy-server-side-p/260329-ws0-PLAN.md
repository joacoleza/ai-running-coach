<objective>
Fix three issues: (1) server-side plan normalization that redistributes training days to correct calendar weeks, (2) pre-computed calendar in system prompt replacing get_week_dates tool reliance, (3) newline preservation in user message storage.

Purpose: Claude sometimes places training days in wrong weeks (e.g. a Wednesday lands in the wrong week object). The per-week `normalizeWeekDays` cannot fix this — it only fills/drops days within each week's Mon-Sun range. A global normalization pass is needed that recomputes which week each day belongs to based on its actual date. Additionally, pre-computing the calendar in the system prompt eliminates tool-call latency and date errors from tool misuse. Newline stripping in stored messages loses formatting.

Output: Corrected plan normalization, pre-computed 26-week calendar in prompt, preserved newlines in chat storage.
</objective>
