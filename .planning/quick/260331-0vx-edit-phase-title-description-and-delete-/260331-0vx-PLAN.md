**Create `api/src/functions/planPhases.ts`** with two Azure Functions HTTP handlers following the exact pattern in `planDays.ts`:

1. **`patchPhase`** — `PATCH /api/plan/phases/{phaseIndex}`
   - `requirePassword` auth check
   - `phaseIndex` param: 0-based integer, validated (`isNaN`, `< 0`, `!isInteger`)
   - Body accepts `{ name?: string; description?: string }` — at least one must be provided, else 400
   - Finds active/onboarding plan via `{ status: { $in: ['active', 'onboarding'] } }`
   - Validates `phaseIndex < plan.phases.length`, else 404
   - Uses `$set` with direct field paths: `phases.{phaseIndex}.name` and/or `phases.{phaseIndex}.description` + `$currentDate: { updatedAt: true }`
   - Returns 200 with `{ plan: result }`

2. **`deleteLastPhase`** — `DELETE /api/plan/phases/last`
   - `requirePassword` auth check
   - Finds active/onboarding plan
   - If `plan.phases.length <= 1`, return 400 `{ error: 'Cannot delete the only phase' }`
   - Check the last phase for any completed days: `lastPhase.weeks.flatMap(w => w.days).some(d => d.completed)` — if any completed, return 409 `{ error: 'Cannot delete a phase with completed days' }`
   - Uses `$pop: { phases: 1 }` to remove the last element + `$currentDate: { updatedAt: true }`
   - Returns 200 with `{ plan: result }`
