<objective>
Fix run:update-insight XML tag leaking into chat display and run insight field.

Purpose: When the AI coach responds to a run feedback request, it may emit self-closing XML tags (`<run:update-insight .../>`, `<app:navigate .../>`, etc.) in `accumulatedText`. These tags are correctly stripped from the CoachPanel display by `useChat.ts`, but `RunDetailModal.tsx` passes the raw `accumulatedText` directly to `updateRun({ insight: responseText })` without stripping — so the raw XML leaks into MongoDB and is then displayed in the run detail modal.
Output: Insight saved to DB is clean plain text with all self-closing XML tags removed.
</objective>

  <action>
In `handleGetFeedback` (around line 124-132), after `const responseText = await sendMessage(message)`, strip all self-closing XML tags from `responseText` before passing to `updateRun`.

Current code:
```typescript
const responseText = await sendMessage(message);
if (responseText && run._id) {
  try {
    const updated = await updateRun(run._id, { insight: responseText });
```

Change to:
```typescript
const responseText = await sendMessage(message);
if (responseText && run._id) {
  try {
    const cleanInsight = responseText.replace(/<[^>]+\/>/g, '').trim();
    const updated = await updateRun(run._id, { insight: cleanInsight });
```

This is the same stripping pattern used for `progressFeedback` in `PATCH /api/plan` (per CLAUDE.md architecture note). The `<[^>]+\/>` regex strips all self-closing XML tags including `<run:update-insight .../>`, `<app:navigate .../>`, and any other agent command tags that may appear in the raw accumulated response text.
