# Quick Task 260423-u59: Fix run:update-insight XML tag visible in insight

**One-liner:** Strip self-closing agent XML tags from coach `responseText` before saving run insight to MongoDB, matching the same pattern already used for `progressFeedback`.

## What Was Done

### Task 1: Strip XML tags in RunDetailModal.handleAddFeedback

In `handleAddFeedback`, the raw `responseText` returned by `sendMessage()` was being passed directly to `updateRun({ insight: responseText })`. Since Claude's response may include self-closing agent command tags (`<run:update-insight .../>`, `<app:navigate .../>`, etc.), these leaked into MongoDB and were then displayed in the run detail modal.

Fixed by adding one line between `sendMessage` return and `updateRun` call:

```typescript
const cleanInsight = responseText.replace(/<[^>]+\/>/g, '').trim();
const updated = await updateRun(run._id, { insight: cleanInsight });
```

This is the same regex used in `PATCH /api/plan` for `progressFeedback` (documented in CLAUDE.md).
