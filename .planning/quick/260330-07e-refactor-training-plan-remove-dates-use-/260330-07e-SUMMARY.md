# Quick Task 260330-07e Summary

Label-based training plan model: replaced all calendar dates with globally sequential week numbers and per-week A-G day labels across all API, frontend, system prompt, and test layers.

## What Was Done

### Task 1 — API types and planUtils
- `PlanDay`: removed `date: string`, added `label: string`
- `PlanWeek`: removed `startDate: string`
- `assignPlanStructure(phases)`: assigns global sequential `weekNumber` and A-G `label` to non-rest days; rest days get `label: ""`
- Removed: `getWeekDates`, `normalizeWeekDays`, `normalizePlanPhases`
- `planUtils.test.ts` rewritten for new function

### Task 2 — API routes
- Routes changed from `plan/days/{date}` to `plan/days/{week}/{day}`
- Validation: `week` must be positive integer, `day` must match `/^[A-G]$/`
- MongoDB arrayFilters: `[{ 'week.weekNumber': week }, { 'day.label': dayParam }]`
- Removed reschedule (`newDate`) logic from patchDay
- addDay uses `{ weekNumber, label, type, ... }` — no date validation
- `plan.ts` and `chat.ts`: import `assignPlanStructure` instead of `normalizePlanPhases`
- `chat.ts` synthetic plan-state injection uses "Week N Day X" format, sorts by weekNumber + label
- `planDays.test.ts` fully rewritten

### Task 3 — System prompt
- Removed entire 26-week pre-computed calendar block (~50 lines)
- Removed `currentDate` parameter from `buildSystemPrompt`
- Updated `plan:update` and `plan:add` tag docs to use `week="N" day="X"` format
- Training plan JSON example: days use `label` not `date`, weeks have no `startDate`
- Current Training Schedule section uses "Week N Day X" format
- `prompts.test.ts` rewritten (23 tests)

### Task 4 — Frontend components, hooks, tests
- `DayRow.tsx`: removed date/reschedule UI (~150 lines), added `weekNumber` prop, displays `Day {label}` statically
- `PlanView.tsx`: AddDayForm uses A-G label buttons instead of date picker; `existingLabels` replaces `existingDates`; sorts by label; `hasAvailableLabel` is `activeDays.length < 7`
- `useChat.ts`: PATCH calls `/api/plan/days/${attrs.week}/${attrs.day}`; POST sends `{ weekNumber, label, ... }`; removed `currentDate` from request bodies
- `planToMarkdown.ts`: uses `Day ${day.label}` format
- All 9 test files updated with label-based fixtures

### Task 5 — Integration and cleanup
- `plan.test.ts`: updated fixtures to label-based, removed 7-day normalization tests and startDate correction test
- Grepped all production code — no stale `day.date` or `.startDate` references remain
- `CLAUDE.md`: updated architecture decisions for day operations, system prompt, DayRow, PlanView, plan:update/add format, synthetic context injection, + Add day button
