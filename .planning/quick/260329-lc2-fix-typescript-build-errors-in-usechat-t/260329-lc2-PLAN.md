# Quick Task 260329-lc2: Fix TypeScript TS2769 Build Errors

## Root Cause

`mockFetch.mock.calls` is typed as `any[][]`. The callbacks at lines 77, 101, and 136 destructured the first element using a tuple type `[string]`, which TypeScript rejects because `any[]` may have fewer than 1 element. Azure SWA's Oryx CI runs `tsc -b && vite build` — when this fails, Oryx skips building and uploads the entire `web/` folder including `node_modules`, exceeding the 262MB limit.

## Tasks

### Task 1: Fix tuple type destructuring in useChat.trainingPlan.test.ts

**Files:** `web/src/__tests__/useChat.trainingPlan.test.ts`

Change all `([url]: [string])` occurrences to `([url]: string[])` at lines 77, 101 (×2), 136.

### Task 2: Document npm run build requirement in CLAUDE.md

**Files:** `CLAUDE.md`

Add bullet to Testing & Documentation section explaining that `npm run build` in `web/` is mandatory before committing, with rationale (SWA size limit when Oryx can't build).
