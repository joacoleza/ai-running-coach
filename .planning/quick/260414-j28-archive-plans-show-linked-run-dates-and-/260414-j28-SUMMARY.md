# Quick Task 260414-j28 Summary

**One-liner:** Archive plan view now uses PlanView in readonly mode with linkedRuns from backend, enabling clickable run dates that open RunDetailModal.

## What Was Built

### Task 1 — GET /api/plans/archived/:id returns linkedRuns

`api/src/functions/planArchive.ts` — the `getArchivedPlan` handler now fetches all runs linked to the archived plan via `planId`, builds a `Record<string, unknown>` keyed by `"weekNumber-dayLabel"`, and returns `{ plan, linkedRuns }` instead of just `{ plan }`. Mirrors the exact pattern in `GET /api/plan`.

### Task 2 — ArchivePlan refactored to PlanView + RunDetailModal

`web/src/pages/ArchivePlan.tsx`:
- Replaced `ReactMarkdown` + `planToMarkdown()` with `<PlanView readonly={true} />` — same component used by TrainingPlan
- Added `linkedRuns` state (`Map<string, Run>`) populated from API response
- Added `open-run-detail` window event listener that sets `selectedRunId`
- Added `selectedRunId` effect that fetches the run and sets `selectedRun`
- Renders `<RunDetailModal>` when `selectedRun` is set — onUpdated/onDeleted handle state updates/cleanup
- Scroll-to-week still works via `document.getElementById('week-N')` since PlanView now emits those IDs

`web/src/components/plan/PlanView.tsx`:
- Added `id={`week-${week.weekNumber}`}` to week h3 — enables scroll targeting from ArchivePlan and TrainingPlan

`web/src/__tests__/ArchivePlan.test.tsx`:
- Removed `react-markdown` and `remark-gfm` mocks (no longer used)
- Added lightweight stubs for `PlanView` and `RunDetailModal`
- Updated `mockFetchWithMessages` to handle `/api/runs/` fetches
- Updated existing tests to use `data-testid="plan-view"` instead of `"markdown"`
- Added new test: `open-run-detail event fetches run and shows RunDetailModal`
