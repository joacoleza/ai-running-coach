# Plan 03.3-02: Run Modal Unification + Unlink Feature

## What Was Built

- **POST /api/runs/:id/unlink endpoint** — clears `planId`, `weekNumber`, `dayLabel` from the run and marks the plan day `completed: false` via arrayFilters
- **`unlinkRun()` client function** — exported from `useRuns.ts`, calls `POST /api/runs/:id/unlink`
- **"Unlink from plan" button in RunDetailModal** — two-step confirm (click → confirm/cancel), dispatches `plan-updated` event after success
- **Notes textarea rows=4** — in both `RunDetailModal` and `RunEntryForm` (was rows=2)
- **Mobile modal height** — `RunDetailModal` outer div uses `max-h-[85dvh]` instead of `max-h-[90vh]`
- **Mobile Safari layout fixes** — FilterPanel grids changed to `grid-cols-1 sm:grid-cols-2`, log run modal uses `items-start overflow-y-auto` + `my-auto`, CSS global 16px font-size rule already in `index.css`
- **`plan:unlink` XML tag in useChat.ts** — stripped from display during streaming and on MongoDB load, processed in `applyPlanOperations` (fetches plan → finds linked run by week+day → calls unlink API)

## Key Files

- `api/src/functions/runs.ts` — POST /api/runs/:id/unlink handler
- `web/src/hooks/useRuns.ts` — `unlinkRun()` exported function
- `web/src/components/runs/RunDetailModal.tsx` — unlink UI, rows=4, max-h-[85dvh]
- `web/src/components/runs/RunEntryForm.tsx` — rows=4
- `web/src/pages/Runs.tsx` — responsive FilterPanel grids, log modal scroll fix
- `web/src/hooks/useChat.ts` — plan:unlink tag parsing and processing
