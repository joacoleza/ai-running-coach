Replace the current two-section approach with a single redesigned feedback panel that always renders when `hasActivePlan && plan` is true:

```tsx
{/* Coach Feedback Panel */}
{hasActivePlan && plan && (
  <div className="mt-2 mb-4 border border-gray-200 rounded-lg overflow-hidden">
    {/* Panel header — always visible */}
    <div className="flex items-center justify-between p-3 bg-gray-50">
      <span className="text-sm font-medium text-gray-700">Coach Feedback</span>
      <button
        onClick={() => { void handleGetFeedback(); }}
        disabled={isRequestingFeedback}
        className="cursor-pointer text-xs text-blue-600 hover:text-blue-800 underline disabled:opacity-50 disabled:cursor-not-allowed"
      >
        {isRequestingFeedback
          ? 'Requesting...'
          : plan.progressFeedback
          ? 'Refresh feedback'
          : 'Get plan feedback'}
      </button>
    </div>

    {/* Expandable content — only renders when feedback exists */}
    {plan.progressFeedback && (
      <>
        <button
          onClick={() => setFeedbackExpanded(!feedbackExpanded)}
          className="cursor-pointer w-full flex items-center justify-between px-3 py-1.5 border-t border-gray-100 bg-gray-50 text-xs text-gray-500 hover:bg-gray-100"
        >
          <span>{feedbackExpanded ? 'Hide' : 'Show'}</span>
          <span>{feedbackExpanded ? '▲' : '▼'}</span>
        </button>
        {feedbackExpanded && (
          <div className="p-3 text-sm text-gray-700">
            <div className="space-y-3">
              {plan.progressFeedback.split('\n\n').map((paragraph, i) => (
                <p key={i}>{paragraph}</p>
              ))}
            </div>
          </div>
        )}
      </>
    )}
  </div>
)}
```

Notes on the implementation:
- `feedbackExpanded` state is auto-set to `true` in `handleGetFeedback` after new feedback is saved (already in existing code — keep this).
- When `plan.progressFeedback` is falsy, the panel shows ONLY the header row (no collapse toggle, no content area).
- When feedback exists but `feedbackExpanded` is false, the header shows, then a "Show ▼" toggle row, then no content.
- When `feedbackExpanded` is true, content renders below the toggle.
- Paragraph spacing: split on `\n\n` and render each paragraph as `<p>` inside `space-y-3` div. This matches chat spacing without ReactMarkdown (simpler, faster, same visual result). Keep ReactMarkdown import if used elsewhere, but this panel no longer needs it.
- The `handleGetFeedback` function body stays the same (sends message, saves progressFeedback, refreshes plan).

Remove the `<button>Get plan feedback</button>` from inside the objective blue box. The objective box becomes:
```tsx
{plan.objective && (
  <div className="mb-4 p-4 bg-blue-50 rounded-lg border border-blue-100">
    <div className="flex flex-wrap items-baseline gap-x-2">
      <span className="font-semibold capitalize">{plan.objective.replace('-', ' ')}</span>
      {plan.targetDate && <span className="text-gray-600">Target: {plan.targetDate}</span>}
    </div>
  </div>
```

```tsx
{!isEditing && !isSaving && (
```
