# Phase 03.3 Plan 01: Scroll and Cross-Navigation Fixes Summary

One-liner: Scroll/nav UX improvements ‚Äî auto-scroll to last completed day, clickable run dates open RunDetailModal, plan badge in modal navigates back to day row, CoachPanel scrolls to bottom on mobile open.

## What Was Built

### Task 1: Auto-scroll TrainingPlan + clickable run date in DayRow (commit 3be5a26)

**TrainingPlan.tsx:**
- Added `useRef`, `useEffect`, `useState` imports
- Added `Run` type and `RunDetailModal` component imports
- Moved `hasActivePlan` computation before hooks (was after early return ‚Äî would cause React hooks order violation)
- Added `lastCompletedRef = useRef<HTMLDivElement | null>(null)` and `dayRefsMap = useRef<Map<string, HTMLDivElement>>(new Map())`
- `useEffect` on `[hasActivePlan]` fires once when plan loads ‚Äî calls `scrollIntoView({ behavior: 'smooth', block: 'center' })` on `lastCompletedRef.current`
- `useEffect` listener for `open-run-detail` custom event ‚Äî sets `selectedRunId`
- `useEffect` on `[selectedRunId]` ‚Äî fetches run from `/api/runs/${selectedRunId}` and sets `selectedRun`
- `useEffect` listener for `navigate-to-day` custom event ‚Äî looks up element in `dayRefsMap` and calls `scrollIntoView`
- `RunDetailModal` rendered conditionally when `selectedRun` is set; `onDeleted` also calls `refreshPlan()`
- `PlanView` receives `lastCompletedDayRef` and `dayRefsMap` props

**DayRow.tsx:**
- Run date changed from `<span>` to `<button>` ‚Äî dispatches `open-run-detail` custom event with `{ runId: linkedRun._id }`

**PlanView.tsx:**
- Added `React` default import (required for `React.MutableRefObject` type assertion)
- `PlanViewProps` extended with `lastCompletedDayRef?` and `dayRefsMap?` props
- `lastCompletedKey` computed inside component body by scanning all phases/weeks/days
- Each DayRow wrapped in `<div>` with callback ref that:
  - Sets `lastCompletedDayRef.current` if this is the last completed day
  - Registers/deregisters element in `dayRefsMap`

### Task 2: RunDetailModal navigation + CoachPanel scroll on open (commit 36a0f9d)

**RunDetailModal.tsx:**
- Added `useNavigate` import from `react-router-dom`
- `const navigate = useNavigate()` at component top
- Plan link badge changed from `<span>` to `<button>`:
  - `onClick`: calls `onClose()`, then `navigate('/plan')`, then `setTimeout(150ms)` dispatches `navigate-to-day` event with `{ weekNumber, dayLabel }`

**CoachPanel.tsx:**
- Added second `useEffect` with `[isOpen]` dependency
- When `isOpen` is true: sets `messagesContainerRef.current.scrollTop = scrollHeight`
- Ensures latest message is visible when mobile user opens panel via FAB

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] hasActivePlan referenced before declaration**
- **Found during:** Task 1
- **Issue:** Plan had `hasActivePlan` defined after the `if (isLoading) return` early return, but the scroll `useEffect` depended on it ‚Äî this would cause a reference-before-declaration error (hooks must be called before any conditional return)
- **Fix:** Moved `const hasActivePlan = ...` computation above all `useEffect` calls, before the early return
- **Files modified:** `web/src/pages/TrainingPlan.tsx`
- **Commit:** 3be5a26
