Rewrite the `loadRuns` / filter / infinite-scroll logic in `web/src/pages/Runs.tsx` to eliminate three stale-closure symptoms: wrong sort order, inflated count ("Showing 43 of 27 runs"), and broken date filter.

Root cause: `loadRuns` is a `useCallback` that closes over `offset`, `dateFrom`, `dateTo`, `distanceMin`, `distanceMax`. The IntersectionObserver captures a stale `loadRuns` reference and fires with old values after filter resets. The filter `useEffect` (line 183) omits `loadRuns` from its dep array, so filter changes call the old closure.

**Fix approach ŌĆö use a ref for offset and pass filters as params:**

1. Replace `const [offset, setOffset] = useState(0)` with `const offsetRef = useRef(0)`.

2. Rewrite `loadRuns` to accept filter params directly instead of reading them from closure:
```tsx
const loadRuns = useCallback(
  async (reset: boolean, filters: { dateFrom?: string; dateTo?: string; distanceMin?: number; distanceMax?: number }) => {
    const currentOffset = reset ? 0 : offsetRef.current;
    if (reset) {
      setIsLoading(true);
      offsetRef.current = 0;
    } else {
      setIsLoadingMore(true);
    }
    try {
      const result = await fetchRuns({
        limit: PAGE_SIZE,
        offset: currentOffset,
        ...filters,
      });
      setRuns(prev => reset ? result.runs : [...prev, ...result.runs]);
      setTotal(result.total);
      offsetRef.current = reset ? result.runs.length : currentOffset + result.runs.length;
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Failed to load runs');
    } finally {
      setIsLoading(false);
      setIsLoadingMore(false);
    }
  },
  [] // no dependencies ŌĆö filters come as params, offset is a ref
);
```

3. Create a helper to build the current filters object from state:
```tsx
const currentFilters = useCallback(() => ({
  dateFrom: dateFrom || undefined,
  dateTo: dateTo || undefined,
  distanceMin: distanceMin ? parseFloat(distanceMin) : undefined,
  distanceMax: distanceMax ? parseFloat(distanceMax) : undefined,
}), [dateFrom, dateTo, distanceMin, distanceMax]);
```

4. Update the filter/mount `useEffect`:
```tsx
useEffect(() => {
  void loadRuns(true, currentFilters());
}, [loadRuns, currentFilters]);
```

5. Update the IntersectionObserver `useEffect` to use `currentFilters`:
```tsx
useEffect(() => {
  const el = sentinelRef.current;
  if (!el) return;
  const obs = new IntersectionObserver((entries) => {
    if (entries[0]?.isIntersecting && !isLoadingMore) {
      // Check total vs offset at call time via ref
      void loadRuns(false, currentFilters());
    }
  });
  obs.observe(el);
  return () => obs.disconnect();
}, [isLoadingMore, loadRuns, currentFilters]);
```

6. In the IntersectionObserver callback, add a guard: only load more if `offsetRef.current < total`. Since `total` is state and may be stale in the observer, store it in a ref too (`totalRef`), updating it alongside `setTotal`. Guard: `if (entries[0]?.isIntersecting && offsetRef.current < totalRef.current && !isLoadingMore)`.

7. Update the "Log a run" modal `onSave` callback to also increment `offsetRef.current` when prepending the new run, and the `onDeleted` callback to decrement it.
