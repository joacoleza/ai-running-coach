**2. Create `api/src/functions/runs.ts`** with these endpoints:

**`POST /api/runs` (createRun)** — route: `'runs'`, method: POST
- Body: `{ date, distance, duration, avgHR?, notes?, weekNumber?, dayLabel?, planId? }`
- Compute `pace` from distance and duration: parse duration string to total minutes, divide by distance. Store as decimal (e.g. 5.5 = 5:30/km).
- If `weekNumber` AND `dayLabel` provided: find active/onboarding plan, verify the day exists and is not already completed, mark the day as `completed: true` using the same `arrayFilters` pattern from planDays.ts (filter by `weekNumber` and `label`), set `planId` on the run to the plan's `_id`.
- Insert run into `runs` collection with `createdAt: new Date(), updatedAt: new Date()`.
- Return 201 with the created run document.
- Return 400 if date/distance/duration missing. Return 409 if day already completed.

**`GET /api/runs` (listRuns)** — route: `'runs'`, method: GET
- Query params: `limit` (default 20), `offset` (default 0), `dateFrom`, `dateTo`, `distanceMin`, `distanceMax`, `durationMin`, `durationMax`
- Build MongoDB filter from query params (date range: `$gte`/`$lte`, distance: `$gte`/`$lte`).
- Sort by `date: -1` (newest first), then `createdAt: -1`.
- Apply `skip(offset)` and `limit(limit)`.
- Return 200 with `{ runs: Run[], total: number }` (total = count without skip/limit for frontend infinite scroll).

**`GET /api/runs/unlinked` (getUnlinkedRuns)** — route: `'runs/unlinked'`, method: GET
- IMPORTANT: Register this route BEFORE the `runs/{id}` route so it doesn't match as an ID.
- Filter: `{ planId: { $exists: false } }` OR `{ planId: null }`.
- Sort by `date: -1`.
- Return 200 with `{ runs: Run[] }`.

**`GET /api/runs/{id}` (getRun)** — route: `'runs/{id}'`, method: GET
- Find by `_id: new ObjectId(id)`.
- Return 200 with run, or 404.

**`PATCH /api/runs/{id}` (updateRun)** — route: `'runs/{id}'`, method: PATCH
- Body: any subset of `{ date, distance, duration, avgHR, notes, insight }`.
- If distance or duration changes, recompute `pace`.
- Update `updatedAt: new Date()`.
- Return 200 with updated run, or 404.

**`DELETE /api/runs/{id}` (deleteRun)** — route: `'runs/{id}'`, method: DELETE
- Find the run first. If `run.planId` exists, return 409 with `{ error: 'Cannot delete a linked run. Undo the training plan day first.' }`.
- Otherwise delete and return 204.

**`POST /api/runs/{id}/link` (linkRun)** — route: `'runs/{id}/link'`, method: POST
- Body: `{ weekNumber: number, dayLabel: string }`
- Find the run by ID (404 if not found).
- Find active/onboarding plan. Verify day exists and is NOT already completed. Return 409 if already completed.
- Update the run: set `planId`, `weekNumber`, `dayLabel`, `updatedAt`.
- Mark the plan day as `completed: true` using arrayFilters.
- Return 200 with updated run.

**Helper function `computePace(distance: number, duration: string): number`:**
- Parse duration: if "HH:MM:SS" → hours*60 + minutes + seconds/60. If "MM:SS" → minutes + seconds/60.
- Return totalMinutes / distance (minutes per unit). Round to 2 decimal places.

**2. Add `PATCH /api/plan` handler in `plan.ts`** for updating progressFeedback:

Register a new handler:
```typescript
app.http('patchPlan', {
  methods: ['PATCH'],
  authLevel: 'anonymous',
  route: 'plan',
  handler: async (req, context) => {
    const denied = await requirePassword(req);
    if (denied) return denied;
    const body = await req.json() as { progressFeedback?: string };
    if (body.progressFeedback === undefined) {
      return { status: 400, jsonBody: { error: 'No updatable fields provided' } };
    }
    const db = await getDb();
    const result = await db.collection('plans').findOneAndUpdate(
      { status: { $in: ['active', 'onboarding'] } },
      { $set: { progressFeedback: body.progressFeedback, updatedAt: new Date() } },
      { returnDocument: 'after' }
    );
    if (!result) return { status: 404, jsonBody: { error: 'No active plan found' } };
    return { status: 200, jsonBody: result };
  }
});
```

Follow existing error handling patterns. Use try/catch on `req.json()`.
