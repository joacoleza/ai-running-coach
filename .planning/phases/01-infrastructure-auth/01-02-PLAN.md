    Create `staticwebapp.config.json` in the repo root with the complete auth configuration (per D-08, D-09, D-11).

    This implements AUTH-01 (owner-only access) and AUTH-02 (unauthenticated redirect to GitHub login).

    Exact file content:
    ```json
    {
      "routes": [
        {
          "route": "/.auth/login/github",
          "allowedRoles": ["anonymous"]
        },
        {
          "route": "/.auth/logout",
          "allowedRoles": ["anonymous", "authenticated"]
        },
        {
          "route": "/api/*",
          "allowedRoles": ["owner"]
        },
        {
          "route": "/*",
          "allowedRoles": ["owner"]
        }
      ],
      "navigationFallback": {
        "rewrite": "/index.html",
        "exclude": ["/api/*", "/assets/*"]
      },
      "responseOverrides": {
        "401": {
          "redirect": "/.auth/login/github?post_login_redirect_uri=.referrer",
          "statusCode": 302
        }
      },
      "platform": {
        "apiRuntime": "node:22"
      }
    }
    ```

    Key details:
    - `"allowedRoles": ["owner"]` on `/*` and `/api/*` — only users with the `owner` custom role can access anything (AUTH-01)
    - `"responseOverrides"."401"` redirects to GitHub OAuth login page (AUTH-02)
    - `/.auth/login/github` is accessible to `anonymous` so the login page itself works
    - `/.auth/logout` is accessible to `anonymous` and `authenticated` for logout flow
    - `navigationFallback` rewrites to `/index.html` for SPA routing (React Router)
    - `platform.apiRuntime` set to `node:22` for Azure Functions

    IMPORTANT: Do NOT use `"allowedRoles": ["authenticated"]` — that would let ANY GitHub user in. The `owner` role is assigned post-deploy via `az staticwebapp users update` (see Plan 03).
