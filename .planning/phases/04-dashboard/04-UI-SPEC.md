## Composition & Layout

### Dashboard Page Structure

**Header section:**
- Page title "Dashboard" (28px, bold, gray-900)
- Filter selector row below title

**Filter row (sticky or inline):**
- Label: "Filter by:" (12px, medium, gray-600)
- Button group: preset tabs or dropdown button (Current Plan / Last 4 weeks / Last 8 weeks / Last 3 months / Last 12 months / This year / All time)
- Current selection indicated by active state (filled bg, bold text) vs inactive (gray text, gray border)
- Responsive: full-width button stack on mobile (≤767px), inline buttons on desktop (md+)

**Stats cards row:**
- 4 cards in a row: Total Distance, Total Runs, Total Time, Adherence %
- Each card: white background, subtle border (gray-200), rounded-lg, p-4
- Card structure: label (12px, medium, gray-600) above, value (24px, bold, gray-900) below
- Adherence % card: cursor-pointer, hover:bg-gray-50 (indicates clickable)
- Responsive: grid-cols-1 on mobile, grid-cols-2 md:grid-cols-4 on desktop

**Charts section:**
- Weekly Volume chart (bar chart, Recharts)
  - Y-axis label: "Distance (km)"
  - X-axis: week starting dates
  - Bar color: green-500 (active), gray-300 (zero/no data)
  - Height: 300px minimum
- Pace Trend chart (line chart, Recharts)
  - Y-axis label: "Pace (min/km)"
  - X-axis: week starting dates
  - Line color: blue-500, markers at each point
  - Height: 300px minimum
- Charts responsive: full-width on mobile and desktop, stacked vertically on mobile (≤767px), side-by-side on desktop (md+, recommended but Claude's discretion)

**Empty states:**
- No active plan + "Current Plan" filter selected:
  - Centered content, page background
  - Icon: activity/calendar (Lucide) or dash pattern
  - Heading: "No active training plan" (20px, semibold, gray-900)
  - Body: "Create a new plan with your coach to get started." (14px, normal, gray-600)
  - CTA button: "Start Planning" → navigates to `/plan` or opens CoachPanel
- No runs in selected range:
  - Centered content
  - Heading: "No runs yet" (20px, semibold, gray-900)
  - Body: "Log your first run from the Training Plan or Runs page." (14px, normal, gray-600)

### Archived Plan Page Updates

**CoachPanel (readonly mode on archived plans):**
- Same position and layout as active plan: right column on desktop (md+, fixed or sticky), FAB-triggered full-screen overlay on mobile
- Visual indicator of readonly mode:
  - Panel label/title: "Plan History" (instead of chat bubble icon or "Coach")
  - Text color: gray-600 (muted vs active coach's gray-900)
  - No input field, no send button — messages display only
  - Message background: same as active (gray-100 for assistant, gray-50 for user)
- Readonly state enforced: `readonly={true}` prop passed to CoachPanel
- FAB on mobile: same position (bottom-6 right-6), visual indicator of readonly (different color: gray-500 vs the active coach's default color, or "History" icon instead of message bubble)

## Interactions & States

### Filter Selector

**States:**
- Default (on load): "Current Plan" filter active
- Hover: bg-gray-100, cursor-pointer
- Active: filled bg (gray-200), bold text
- Inactive: gray text (gray-600), transparent bg
- Disabled (when no active plan and "Current Plan" selected): appears but cannot select other filters (all filters available always per CONTEXT.md D-07)

**Behavior:**
- Click a preset → fetch runs with that date range / planId filter
- Fetch runs via `fetchRuns()` with appropriate `dateFrom`, `dateTo`, `planId` parameters
- Show loading spinner while fetching
- Update all stats and charts once fetch completes

### Stat Cards

**Adherence % card interaction:**
- Hover: bg-gray-50, cursor-pointer, subtle underline or highlight
- Click: navigate to `/plan` (activates PlanView scroll to top)
- Visual indicator of clickability: slight elevation (shadow) on hover

### Charts

**Weekly Volume bar chart:**
- Bar hover: color darkens to green-600, shows tooltip with week start date and distance
- Responsive: bars scale width based on screen size
- Empty weeks: render with gray-300 bars at 0 height or omit entirely (Claude's discretion)

**Pace Trend line chart:**
- Line hover: shows tooltip with week start date and pace
- Marker hover: highlights point, shows exact pace value
- No bars (line chart only)
- Empty weeks: line breaks or shows interpolated point (Claude's discretion)

### Readonly CoachPanel (Archived Plans)

**States:**
- Messages render in same format as active coach (user message left, assistant message right)
- Scrolling: scroll to bottom on open (same as active)
- No input, no send → readonly interaction
- No streaming (messages are historical)

**Visual indicators:**
- Panel title: "Plan History" (gray-600, smaller or muted font vs "Coach")
- Close/collapse mechanism: X button (same as active) on mobile, fixed position on desktop
- FAB icon on mobile: "history" or "file-text" icon (gray-500) vs active coach message bubble icon

## Notes for Executor (gsd-executor)

1. **Filter presets implementation:** Use a date utility to compute date ranges for each preset (e.g., "Last 4 weeks" = today - 28 days). Current Plan filter requires active plan check.

2. **Stats calculation:**
   - Total Distance: sum of all `run.distance` values in filtered set
   - Total Runs: count of runs in filtered set
   - Total Time: sum of all `run.duration` parsed as minutes, format as "XhYm"
   - Adherence %: `(completed linked runs / total non-rest days in plan window) * 100` for "Current Plan" filter; for date-based filters, use `(completed linked runs in range / total planned days in range) * 100`

3. **Chart data preparation:**
   - Group runs by week (use ISO week or date math)
   - Weekly Volume: sum distances per week
   - Pace Trend: average pace per week (exclude zero/rest weeks)
   - X-axis label: week starting Monday (format "Apr 7" or similar)

4. **Readonly CoachPanel:** Pass `readonly={true}` prop, messages from `linkedRuns` or archived plan chat history, no message input handler.

5. **Mobile responsiveness:** Charts and filters test on iPhone 375px viewport (Tailwind md breakpoint is 768px).

6. **Charting library:** Recharts is pre-recommended. If executor finds a better fit, document decision in PHASE-CONTEXT.md update.
