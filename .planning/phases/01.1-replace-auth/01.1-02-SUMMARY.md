## What Was Built

**PasswordPage component** (`web/src/pages/PasswordPage.tsx`):
- Password form with centered card layout (Tailwind — `min-h-screen flex items-center justify-center`)
- Validates against `/api/health` with `X-App-Password` header
- Stores password in `localStorage["app_password"]` on 200 response
- Shows "Wrong password" on 401, "Service locked. Contact administrator." on 503
- Calls `onSuccess()` prop to hand off to App.tsx on success

**App.tsx auth gate** (`web/src/App.tsx`):
- `authenticated` state initialized from `localStorage.getItem('app_password')`
- Renders `<PasswordPage>` when not authenticated — entire router/AppShell hidden behind gate
- Global `window.fetch` interceptor in `useEffect` — any 401 from any API call clears localStorage and sets `authenticated = false`
- Interceptor properly restored on unmount via cleanup function

**Sidebar logout** (`web/src/components/layout/Sidebar.tsx`):
- Replaced `<a href="/.auth/logout">` with `<button type="button">`
- On click: `localStorage.removeItem('app_password')` then `window.location.href = '/'`
- Full page reload triggers App.tsx to re-evaluate auth state and render PasswordPage
- Visual styling preserved (same className, icon, label)

**SWA config** (`web/staticwebapp.config.json`):
- Removed all GitHub OAuth routes (`/.auth/login/github`, `/.auth/invitations/accept`, `/.auth/logout`)
- Removed `unauthorized.html` route
- Removed owner role from `/*` and `/api/*` — replaced with single `"anonymous"` wildcard
- Removed `responseOverrides` (401 redirect to GitHub login, 403 redirect to unauthorized)
- Kept `navigationFallback` and `platform.apiRuntime: node:22`
