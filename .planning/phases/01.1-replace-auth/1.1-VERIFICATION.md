### Human Verification Required

The following items cannot be verified by static analysis and require a running browser:

#### 1. Password Prompt Renders on First Visit

**Test:** Open the app URL in a browser with no `app_password` in localStorage (or incognito).
**Expected:** A centered card with "AI Running Coach" title and a password input field renders immediately — no GitHub OAuth redirect occurs.
**Why human:** Static analysis confirms the conditional render code path exists; actual browser paint and absence of redirect cannot be asserted statically.

#### 2. Correct Password Unlocks the App

**Test:** Enter the correct password and submit.
**Expected:** The AppShell with sidebar and dashboard renders — password page disappears without a page reload.
**Why human:** Requires live API call returning 200 and React state transition.

#### 3. Wrong Password Shows Error

**Test:** Enter an incorrect password and submit.
**Expected:** "Wrong password" message appears in red below the form; the page does not navigate away.
**Why human:** Requires live API call returning 401 and error state render.

#### 4. Logout Returns to Password Prompt

**Test:** After logging in, click the Logout button in the sidebar.
**Expected:** Page reloads to the password prompt. No GitHub OAuth redirect, no Azure endpoint hit (`/.auth/logout` never called).
**Why human:** `window.location.href = '/'` behavior and absence of external redirects require browser observation.

#### 5. Stored Password Persists Across Refresh

**Test:** Log in successfully, then refresh the page (F5).
**Expected:** App loads directly to the dashboard — password prompt does not appear.
**Why human:** Requires browser with persisted localStorage state across page reload.
