# Phase 02.1 Plan 02: API Endpoints (PATCH day, Archive, Import) + System Prompt Summary

New PATCH day endpoint with MongoDB arrayFilters completed-day guard, three archive endpoints, ChatGPT share URL import endpoint with Claude-based parsing, and system prompt fully updated to hierarchical PlanPhase format with plan:update protocol documentation.

## What Was Built

### Task 1: PATCH /api/plan/days/:date and archive endpoints

**api/src/functions/planDays.ts** — `patchDay` Azure Function handler:
- `PATCH /api/plan/days/{date}` route with requirePassword auth
- Validates date format (YYYY-MM-DD regex) — returns 400 for invalid
- Accepts body fields: `guidelines`, `objective_kind/value/unit`, `completed`, `skipped`
- Uses MongoDB `findOneAndUpdate` with `arrayFilters: [{ 'day.date': date, 'day.completed': false }]` to enforce the completed-day guard (D-15/D-17)
- Returns 404 if day not found or already completed

**api/src/functions/planArchive.ts** — Three handlers:
- `archivePlan` — POST `/api/plan/archive`: findOneAndUpdate active plan to `status: archived`
- `listArchivedPlans` — GET `/api/plans/archived`: returns projected list (_id, objective, goal, createdAt)
- `getArchivedPlan` — GET `/api/plans/archived/{id}`: returns single archived plan by ObjectId

### Task 2: Import endpoint and system prompt update

**api/src/functions/planImport.ts** — `importPlan` Azure Function handler:
- `POST /api/plan/import` validates URL matches `https://chatgpt.com/share/` pattern
- Fetches URL with realistic User-Agent headers
- `extractTextFromHtml()` strips script/style/tag content
- Validates extracted text length (>200 chars) and absence of login walls
- Sends truncated text (15000 chars) to Claude with a training plan parsing system prompt
- Extracts JSON from `<training_plan>` tags in Claude's response
- Archives any existing active plan, inserts new active plan with parsed phases
- Returns 201 with new plan

**api/src/shared/prompts.ts** — Full rewrite of relevant sections:
- Import changed from `PlanSession` to `PlanPhase`
- Function signature: `buildSystemPrompt(summary?, onboardingStep?, phases?: PlanPhase[])`
- Added "Plan Update Command" section documenting `<plan:update>` tag protocol
- Training Plan Format section updated to hierarchical `phases/weeks/days` JSON structure
- Upcoming sessions section replaced with upcoming days from `phases[]` with date/type/objective/guidelines
- Removed all `session_id` / `<app:complete>` references (count: 0)

**api/src/functions/chat.ts** — Minor update to pass `plan.phases` as third arg to `buildSystemPrompt`.
