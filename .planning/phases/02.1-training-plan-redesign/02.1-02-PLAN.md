**Create api/src/functions/planDays.ts:**

Register an Azure Function `patchDay` with `methods: ['PATCH']`, `authLevel: 'anonymous'`, `route: 'plan/days/{date}'`.

Handler logic:
1. Call `requirePassword(req)` — return denied if not null.
2. Extract `date` from `req.params.date`. Validate it matches `YYYY-MM-DD` format with regex `/^\d{4}-\d{2}-\d{2}$/`. Return 400 if invalid.
3. Parse request body as `{ guidelines?: string; objective_kind?: string; objective_value?: string; objective_unit?: string; completed?: string; skipped?: string; }`.
4. Build a `$set` object for MongoDB. Map fields:
   - If `guidelines` present: `$set['phases.$[].weeks.$[].days.$[day].guidelines'] = body.guidelines`
   - If `objective_kind` + `objective_value` + `objective_unit` all present: `$set['phases.$[].weeks.$[].days.$[day].objective'] = { kind: body.objective_kind, value: Number(body.objective_value), unit: body.objective_unit }`
   - If `completed === 'true'`: `$set['phases.$[].weeks.$[].days.$[day].completed'] = true`
   - If `skipped === 'true'`: `$set['phases.$[].weeks.$[].days.$[day].skipped'] = true`
   - Always add `$currentDate: { updatedAt: true }`
5. Use MongoDB `findOneAndUpdate` with:
   - Filter: `{ status: 'active' }` (only the active plan)
   - `arrayFilters: [{ 'day.date': date, 'day.completed': false }]` — enforces D-15/D-17 (no edits to completed days)
   - `returnDocument: 'after'`
6. If result is null, return 404 `{ error: 'Day not found or already completed' }`.
7. Return 200 with `{ plan: result }`.
