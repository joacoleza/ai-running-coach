# Phase 02.1 Plan 01: Data Model Foundation Summary

Replaced flat sessions[] training plan schema with hierarchical phases[]->weeks[]->days[] type system. Removed react-big-calendar entirely and updated both backend and frontend to use the new structure.

## What Was Built

**New type hierarchy (api/src/shared/types.ts):**
- `PlanDay` — individual training day with type (run/rest/cross-train), optional objective, guidelines, completed/skipped flags
- `PlanWeek` — week container with weekNumber, startDate, days[]
- `PlanPhase` — phase container with name, description, weeks[]
- Updated `Plan` interface: `phases: PlanPhase[]`, `objective?`, `targetDate?`, `status: 'onboarding' | 'active' | 'archived'`
- `PlanSession` kept as deprecated export for transition period

**Backend changes (api/src/functions/plan.ts):**
- `generatePlan` now parses `{ phases: PlanPhase[] }` from `<training_plan>` tags instead of flat session array
- Validates phases array has at least one entry
- `createPlan` uses `phases: []` instead of `sessions: []`
- `getPlan` returns `null` for stale plans with `sessions[]` but no `phases[]`
- `createPlan` archives (not discards) displaced onboarding plans

**Frontend changes:**
- `usePlan.ts`: New PlanDay/PlanWeek/PlanPhase/PlanData types matching backend. Removed `updateSession` function.
- `useChat.ts`: PlanData updated to use `phases[]`, status includes `archived`, imports types from usePlan.ts
- `TrainingPlan.tsx`: Replaced PlanCalendar/SessionModal with placeholder showing objective and phase count
- react-big-calendar and @types/react-big-calendar uninstalled (27 packages removed)
