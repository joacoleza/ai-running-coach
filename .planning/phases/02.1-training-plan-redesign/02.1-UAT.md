## Tests

### 1. Training Plan Displays in Hierarchy
expected: Open the Training Plan page. The plan renders as a hierarchy: Phases at the top level, each phase containing weeks, each week containing days (not a flat list or calendar grid). Rest days should not appear. You should see phase names, week numbers, and individual training days with their type (run, cross-train, etc.) and guidelines.
result: pass

### 2. Inline Edit Day Guidelines
expected: Click on a day's guidelines text. It should switch to an editable input field. Type a change, then press Enter or click away (blur). The field saves and reverts to display mode showing the updated text. Pressing Escape should cancel the edit and restore the original text.
result: pass

### 3. Mark a Day as Complete
expected: Click the Complete button on an active training day. The day should show a green checkmark and become read-only (no edit or action buttons visible). The Complete and Skip buttons should disappear for that day.
result: pass

### 4. Skip a Day
expected: Click the Skip button on an active training day. The day should show strikethrough text and become read-only. The action buttons should disappear for that day.
result: pass

### 5. New Plan Button Opens Coach
expected: Click "New Plan" in the action bar. The coach panel should open and the conversation should start (conversational onboarding begins).
result: pass

### 6. Import Button Shows URL Form
expected: Click "Import from ChatGPT" in the action bar. A URL input form should appear with a field for a ChatGPT share URL and a submit button.
result: skipped
reason: No longer a requirement — feature was removed intentionally

### 7. Update Plan Button Opens Coach
expected: Click "Update Plan" in the action bar. The coach panel should open so you can chat with the coach about modifying the plan.
result: pass

### 8. Archive Plan
expected: Click "Close & Archive" in the action bar. The current active plan should be archived. You should be taken to a state where you can start a new plan (or the plan page shows no active plan).
result: pass

### 9. Archive Section in Sidebar
expected: The sidebar shows an "Archive" navigation item (with a file cabinet icon 🗄️). Clicking it navigates to the archive list page.
result: pass

### 10. Archive List Page
expected: The archive page shows a list of previously archived plans. Each item shows the plan's objective (or event type) and an "Archived" status badge. Each item is clickable and links to the archived plan detail.
result: pass

### 11. Archived Plan Readonly View
expected: Click an archived plan from the list. The detail page renders the plan in markdown format (phases, weeks, days as formatted text). There are no edit controls — it is fully read-only. A back link returns you to the archive list.
result: pass

### 12. Coach Can Update Plan Days via Chat
expected: Open the coach panel and ask the coach to update a specific training day (e.g. "Change Monday's run to 8km easy pace"). The coach should respond and the plan on the Training Plan page should update automatically without a page reload.
result: pass
