Archive section added with sidebar nav entry, list page fetching from GET /api/plans/archived, and readonly plan view using planToMarkdown for consistent markdown rendering.

## What Was Built

**Archive list page (`web/src/pages/Archive.tsx`):**
- Fetches archived plans from `GET /api/plans/archived` on mount
- Displays each plan with its objective (or goal.eventType fallback) as the primary label
- Shows an "Archived" status badge satisfying D-24's completion status requirement
- No date range displayed, per D-24 spec
- Each item links to `/archive/:id`

**Archived plan readonly view (`web/src/pages/ArchivePlan.tsx`):**
- Fetches a single archived plan from `GET /api/plans/archived/:id` using `useParams`
- Converts plan to markdown using `planToMarkdown` imported from `web/src/utils/planToMarkdown.ts` (created in Plan 03)
- Renders markdown via `ReactMarkdown` with Tailwind `prose prose-sm` typography (D-25)
- Back navigation link to `/archive`

**Sidebar navigation (`web/src/components/layout/Sidebar.tsx`):**
- Archive nav item added after Runs in `navItems` array (D-23)
- Existing NavLink rendering loop handles it automatically

**App routing (`web/src/App.tsx`):**
- Imported `Archive` and `ArchivePlan`
- Added `/archive` and `/archive/:id` routes before the catch-all `*` route

## Decisions Made

- **Archive status badge**: Since all plans in this list have `status: archived` by definition, displaying a static "Archived" badge on each item satisfies D-24's "completion status" requirement clearly and without redundancy.
- **planToMarkdown reuse**: Rather than a custom archive renderer, the view reuses `planToMarkdown` from Plan 03 so archived plans display in the exact same format as the active plan's export, ensuring consistency.
- **Archive nav icon**: Used 🗄️ (file cabinet) to semantically distinguish archive from active plan calendar (📅) and runs (🏃).

## Deviations from Plan

None - plan executed exactly as written. The only minor adjustment was adding TypeScript type assertions (`as { plans?: ArchivedPlanSummary[] }`) for the fetch responses to satisfy strict TypeScript and avoid `any` type warnings.

## Known Stubs

None — Archive.tsx fetches from real API endpoint. ArchivePlan.tsx fetches and renders real plan data. No placeholder data or hardcoded values.
