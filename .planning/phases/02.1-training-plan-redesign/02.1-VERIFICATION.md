#### 2. Inline Click-to-Edit on Live Plan

**Test:** Navigate to the Training Plan page with an active plan. Click on a day's guidelines text. Verify an input appears. Change the text and press Enter (or blur). Verify the change persists on reload.
**Expected:** Edit field appears on click; saves on blur/Enter; PATCH call goes to backend; value persists.
**Why human:** Integration between DayRow click handler, `usePlan.updateDay`, and the backend requires a live MongoDB instance with a real plan.

#### 3. Coach Panel Plan Update (plan:update tag)

**Test:** With an active plan, open the coach panel, say "mark my [date] run as done" or "change my [date] run guidelines to tempo intervals". Observe the plan view.
**Expected:** The coach responds, the `plan:update` tag is stripped from the displayed message, and the day in the plan view is updated without a page reload.
**Why human:** Requires live Claude API + SSE streaming + PATCH round-trip. The unit tests mock the streaming and PATCH calls.
