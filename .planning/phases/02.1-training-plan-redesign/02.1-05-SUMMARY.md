# Phase 02.1 Plan 05: Wire useChat plan:update and Unit Tests Summary

Final wiring that makes agent-driven plan updates work end-to-end, plus full unit test coverage for all Phase 2.1 features.

## What Was Built

**Task 1: Wire useChat for plan:update tags (web/src/hooks/useChat.ts)**

- Added `parseXmlAttrs(attrString)` exported helper that extracts `key="value"` pairs from an XML attribute string using `matchAll`
- Added `plan:update` tag handling in `sendMessage`'s `payload.done` else branch (non-training_plan path): strips tags from displayed message, PATCHes each `<plan:update date="..." .../>` to `/api/plan/days/:date`, re-fetches plan after all updates
- Added identical `plan:update` handling in `startPlan`'s conversational mode `payload.done` else branch -- ensures plan:update works whether user is in normal chat or conversational onboarding flow
- Added `objective` field to `plan/generate` POST body in both `sendMessage` and `startPlan` (passes `extractedGoal.eventType`)
- Removed `<app:complete session_id="...">` handling and `PATCH /api/sessions/:id` loop -- completion is now handled via `<plan:update date="..." completed="true" />`
- Confirmed `api/src/functions/chat.ts` already passes `plan.phases` to `buildSystemPrompt` (no change needed)
