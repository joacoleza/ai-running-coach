## Decisions Made

- PlanView renders React components directly rather than planToMarkdown + ReactMarkdown for the active view, so inline editing controls can be embedded in the rendered output. planToMarkdown is kept for archive readonly rendering and export.
- open-coach custom window event pattern: TrainingPlan dispatches it, AppShell listens. Avoids prop-drilling or context for this cross-cutting concern.
- DayRow read-only guard uses `readonly || day.completed || day.skipped` — three separate reasons a day could be non-editable all collapsed into one `isReadOnly` boolean.
