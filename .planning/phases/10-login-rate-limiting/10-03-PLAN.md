**LoginPage.tsx — three targeted changes inside `handleSubmit` (lines 52–60):**

1. Fix the 401 branch (currently line 52–54). Replace:
   ```ts
   } else if (response.status === 401) {
     setError('Invalid email or password');
     setPassword('');
   ```
   With:
   ```ts
   } else if (response.status === 401) {
     const data = await response.json() as { error?: string };
     setError(data.error || 'Invalid email or password');
     setPassword('');
   ```

2. Add a 429 branch immediately after the 401 branch, before the 503 branch:
   ```ts
   } else if (response.status === 429) {
     const data = await response.json() as { error?: string };
     setError(data.error || 'Account locked. Try again later.');
     setPassword('');
   ```

3. Leave the 503 branch and the else branch unchanged — they remain for service-unavailable and unexpected errors respectively.

Final else-if chain order must be: 401 → 429 → 503 → else.

2. Add a new test after the 401 tests:
   ```ts
   it('shows lockout message from API body on 429 response', async () => {
     global.fetch = vi.fn().mockResolvedValue({
       ok: false,
       status: 429,
       json: vi.fn().mockResolvedValue({ error: 'Account locked. Try again in 15 minutes.' }),
     })

     render(<LoginPage onTempPassword={mockOnTempPassword} />)
     fireEvent.change(screen.getByLabelText('Email'), { target: { value: 'user@example.com' } })

     fireEvent.click(screen.getByRole('button', { name: /log in/i }))

     await waitFor(() => {
       expect(screen.getByText('Account locked. Try again in 15 minutes.')).toBeInTheDocument()
     })
     // Password field should be cleared after lockout
     expect(screen.getByLabelText('Password')).toHaveValue('')
   })
   ```

3. Add a 429 fallback test to verify the default message when API body lacks `error`:
