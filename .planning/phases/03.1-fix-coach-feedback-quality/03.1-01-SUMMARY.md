# Phase 03.1 Plan 01: Fix Coach Feedback Quality â€” Summary

## One-liner

Stale-closure fix for run insight (sendMessage return value) and XML-stripping for plan progressFeedback using regex replace before PATCH save.

## What Was Built

Fixed two warning-severity integration bugs in the coaching loop:

**COACH-03: RunDetailModal stale closure**
- `handleAddFeedback` was reading from `messages` array after `await sendMessage()`, which captured the stale pre-response snapshot
- Fixed: `sendMessage` now returns the response text directly; code uses `const responseText = await sendMessage(message)` and passes it to `updateRun`
- `messages` removed from `useChatContext()` destructure â€” no longer referenced

**COACH-04: Raw XML in progressFeedback**
- `handleGetFeedback` was saving the raw `responseText` which could contain `<plan:update .../>` and other self-closing XML tags
- Fixed: `cleanedText = responseText.replace(/<[^>]+\/>/g, '').trim()` applied before the `if` guard and before the fetch body
- When stripping leaves only whitespace, the PATCH call is skipped entirely
