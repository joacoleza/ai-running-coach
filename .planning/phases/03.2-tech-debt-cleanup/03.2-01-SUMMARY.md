# Phase 03.2 Plan 01: Remove Dead-Code API Endpoints Summary

Deleted `PATCH /api/sessions/:sessionId` and `POST /api/plan/generate` — two endpoints with no frontend callers that existed as dead weight after architectural shifts.

## What Was Built

Removed two dead API endpoints and their associated tests:

1. **`sessions.ts` deleted** — The `PATCH /api/sessions/:sessionId` endpoint was superseded by `PATCH /api/plan/days/:week/:day` during the training plan redesign (Phase 2.1). It still used the old `sessions[]` data model and had no frontend callers.

2. **`generatePlan` handler removed from `plan.ts`** — The `POST /api/plan/generate` endpoint was superseded by server-side plan saving in `chat.ts` (architecture decision recorded in STATE.md). The client now re-fetches via `GET /api/plan` after receiving `planGenerated: true` from the SSE stream.

3. **Import cleanup** — Removed `sessions.js` import from `index.ts`, removed `assignPlanStructure` and `PlanPhase` imports from `plan.ts` (both were only used by the deleted `generatePlan` handler).

4. **Test cleanup** — Deleted `sessions.test.ts` entirely. Removed all `generatePlan` describe blocks from `plan.test.ts` (21 test cases across 5 describe blocks). Kept all `createPlan`, `getPlan`, `patchPlan`, and `Plan Schema` test cases intact.
