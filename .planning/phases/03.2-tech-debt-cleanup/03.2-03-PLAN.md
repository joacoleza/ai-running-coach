**Part B — Fix act() warning in App.auth.test.tsx:**

The `act()` warning occurs in test 2 ("renders AppShell with Dashboard nav link..."). When `App` renders with `app_password` set in localStorage, it renders `AppShell` → `ChatProvider` → `useChat`, which runs an async `useEffect` on mount that calls `fetch('/api/plan')` then `fetch('/api/messages?planId=...')`. These async state updates after `render()` complete outside `act()`, causing the warning.

Fix: Wrap the `render` call AND await all pending state updates using `waitFor` from `@testing-library/react`. The mock `fetch` already returns `{ ok: true, status: 200 }` but needs a `json()` method to resolve without errors.

**Updated test 2:**
```typescript
it('renders AppShell with Dashboard nav link when app_password is set in localStorage', async () => {
  localStorage.setItem('app_password', 'test-password')
  // Provide a json() method so useChat's fetchPlan doesn't throw
  global.fetch = vi.fn().mockResolvedValue({
    ok: true,
    status: 200,
    json: vi.fn().mockResolvedValue({ plan: null }),
  } as unknown as Response)

  await act(async () => {
    render(<App />)
  })
  // Allow pending async effects (fetchPlan, fetchMessages) to resolve
  await waitFor(() => {
    const dashboardElements = screen.getAllByText('Dashboard')
    expect(dashboardElements.length).toBeGreaterThan(0)
  })
})
```

Add `waitFor` to the import line at the top: `import { render, screen, act, waitFor } from '@testing-library/react'`

This ensures all state updates from `useChat`'s mount effect complete inside `act()` before the assertion runs.
