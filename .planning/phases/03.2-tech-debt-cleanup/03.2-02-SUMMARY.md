# Phase 3.2 Plan 02: Deduplicate SSE Streaming Loop Summary

Extracted a shared `streamChatResponse` helper eliminating ~200 lines of duplicated SSE reader/decoder/buffer/payload logic from `sendMessage` and `startPlan`. Both functions now delegate to the helper; the plan:update/add/phase operation logic was also consolidated into `applyPlanOperations`.

## What Was Built

The `useChat.ts` hook previously contained two independent copies of the SSE streaming loop (lines ~190-462 in `sendMessage`, lines ~532-757 in `startPlan`). Both copies had identical:
- `getReader()` / `TextDecoder` / buffer accumulation
- Line-by-line `data:` prefix stripping + JSON parse
- `payload.text` streaming display with XML tag stripping
- `payload.done` / `payload.error` handling

**Extracted `streamChatResponse`** (module-level, above the hook):
- Takes `Response` + `{ alive?, onText, onDone, onError }` callbacks
- Handles all SSE reading/buffering/parsing
- Returns accumulated text string
- Calls `alive()` guard before each `reader.read()` to support session invalidation

**Extracted `applyPlanOperations`** (useCallback inside hook):
- Handles `plan:update`, `plan:add`, `plan:update-phase`, `plan:delete-phase` tag processing
- Shared between `sendMessage.onDone` and `startPlan.onDone`
- Accepts optional `aliveCheck` for the `startPlan` session guard
