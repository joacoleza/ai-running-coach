## Decisions Made

1. **Playwright webServer uses `npm run start` (not `npx func start` directly)** — `npm run start` runs `prestart` (TypeScript build) first, ensuring the Functions server starts with compiled code. Direct `npx func start` would fail if `dist/` was stale.

2. **`no-commit: true` + manual orphan branch push for coverage badges** — `master` is branch-protected. Using `no-commit: true` prevents the badge action from trying to commit to master. The separate "Push badges to badges branch" step creates an orphan branch and force-pushes only the badges, keeping the badges branch clean.

3. **Single `test` job (not separate unit/e2e jobs)** — E2E tests need MongoDB running. Having one job means MongoDB is started once and shared across API unit tests, web unit tests, and E2E tests, saving CI time and avoiding duplicate MongoDB startup steps.

4. **MongoDB seeded directly in lockout E2E test** — Submitting the wrong password 30 times through the browser would take 30+ seconds and be flaky. Direct `MongoClient.insertOne({ _id: 'lockout', failureCount: 30, blocked: true })` seeds the blocked state instantly and tests the same UI behavior.
