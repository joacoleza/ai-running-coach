## Decisions Made

1. **vi.hoisted() for mock function references in vi.mock() factory** — `vi.mock()` is hoisted to the top of the file by Vitest. Variables declared with `const` are not accessible at hoist time, causing `ReferenceError: Cannot access 'X' before initialization`. Solution: use `vi.hoisted()` to declare mock functions that are accessible when the hoisted `vi.mock()` factory executes.

2. **_resetConnectionForTest() exported unconditionally** — The plan noted it could be guarded by `NODE_ENV === 'test'`. Decided to export unconditionally for simplicity; the function is a no-op in production if never called (it just nulls out the cached client which would be reconnected on next request).

3. **Integration tests use vi.resetModules() + dynamic import** — Each `beforeEach` calls `vi.resetModules()` then `await import('../middleware/auth.js')` to get a fresh module instance. This ensures the MongoClient singleton in auth.ts reconnects to the current mongodb-memory-server URI after each test resets the collection.
