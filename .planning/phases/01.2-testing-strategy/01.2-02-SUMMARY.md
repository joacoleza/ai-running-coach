## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Form submit used wrong query pattern**
- **Found during:** Task 2 RED phase
- **Issue:** Plan instructed `screen.getByRole('form') ?? input.closest('form')!` but the `<form>` element has no accessible role name (no `aria-label` attribute), so `getByRole('form')` threw instead of returning null
- **Fix:** Changed to `input.closest('form')!` directly
- **Files modified:** `web/src/__tests__/PasswordPage.test.tsx`
- **Commit:** `c52a988`

**2. [Rule 1 - Bug] Multiple elements with text "Dashboard"**
- **Found during:** Task 2 RED phase
- **Issue:** Plan instructed `getByText('Dashboard')` but the authenticated App renders Dashboard both in the sidebar nav link span AND in the Dashboard page heading `<h1>`, causing a "Found multiple elements" error
- **Fix:** Changed to `getAllByText('Dashboard')` and checked `length > 0`
- **Files modified:** `web/src/__tests__/App.auth.test.tsx`
- **Commit:** `c52a988`
